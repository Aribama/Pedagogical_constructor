# Цифровой педагогический конструктор


**Цифровой педагогический конструктор** — это сервис для поддержки проектирования учебных занятий, обеспечивающий удобный подбор педагогических приёмов и их адаптацию к предметному содержанию урока с использованием генеративного искусственного интеллекта.

Проект решает проблему сложности ориентирования в большом количестве методических механик и трудности их органичного встраивания в канву конкретного урока.

## Основные возможности

* **Гибкая фильтрация**: поиск приёмов по педагогическим параметрам: предмет, уровень учебной группы, этап занятия, формат работы и др.


* **Интеграция с ИИ**: автоматизированное встраивание выбранных педагогических приёмов в предметное содержание занятия для формирования подробного сценария.


* **Методическая база**: структурированное хранение информации о педагогических механиках, классифицированных под руководством педагога-наставника.


* **Мультиагентный алгоритм**: использование специализированной схемы формирования запросов к ИИ для повышения качества и методической точности планов-конспектов.



## Стек технологий

* **Backend**: Django, Django REST Framework, PostgreSQL.


* **Frontend**: React, Vite, TypeScript, Axios.
* **AI**: Интеграция с API генеративных моделей (LLM) для адаптации приёмов.

## Установка и запуск

### 1. Подготовка Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate  # venv\Scripts\activate для Windows
pip install -r requirements.lock.txt

```

*Создайте файл `.env`, укажите параметры БД и API ключи для ИИ.*

```bash
python manage.py migrate
python3 manage.py createsuperuser # создание СуперПользователя
python manage.py runserver 8000

```

*Далее сервер будет доступен по адресу:*

```bash
http://localhost:8000/
```

### 2. Запуск Frontend

```bash
cd frontend
pnpm install  # или npm install
pnpm dev

```
*Frontend будет доступен по адресу:*

```bash
http://localhost:5173
```
### 3. Создание карточек
*Шаблон:*

```bash
python3 manage.py create_card \
  --owner lessonapp # если есть пользователь lessonapp \
  --kind technique \        # technique | aux_org | aux_team_split | aux_warmup | aux_reflection
  --title "Название карточки" \
  --description "Описание и инструкции, текст в HTML-формате" \
  \
  --duration 5 \
  \
  --activity calm \          # active | calm
  --bloom understand \       # remember | understand | apply | analyze | evaluate | create
  \
  --a1 \                     # 1–4 классы (один или несколько)
  --a2 \                     # 5–8 классы
  --a3 \                     # 9–11 классы
  \
  --individual \             # индивидуальная работа (один или оба)
  --group \                  # групповая работа
  \
  --critical \               # 4K: критическое мышление (один или несколько)
  --creative \               # 4K: креативность
  --communication \          # 4K: коммуникация
  --collaboration \          # 4K: кооперация
  \
  --stage-start \            # начало занятия (один или несколько)
  --stage-core \             # основная часть
  --stage-final \            # завершение
  \
  --public                   # сделать карточку публичной - всегда по умолчанию да
```

*Пример создания карточки:*

```bash
python3 manage.py create_card \
  --owner lessonapp \
  --kind technique \
  --title "Упражнение на концентрацию" \
  --description "<p>Ученики садятся в круг, закрывают глаза и делают дыхательное упражнение на 2 минуты. После обсуждают, что почувствовали.</p>" \
  --duration 5 \
  --activity calm \
  --bloom understand \
  --a1 \
  --a2 \
  --individual \
  --group \
  --critical \
  --creative \
  --communication \
  --collaboration \
  --stage-start \
  --stage-core \
  --public
```

## Особенности архитектуры

Проект объединяет техническую реализацию и методическую экспертизу:

1. **Пользователь** задаёт параметры занятия и выбирает приёмы в интерфейсе.


2. **Система** на основе мультиагентного алгоритма формирует сложный запрос (промпт) к ИИ.


3. **ИИ** генерирует итоговый план-конспект, учитывающий логику и структуру урока при сохранении выбранных механик.
